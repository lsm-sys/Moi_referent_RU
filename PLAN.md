# PLAN.md — план добавления функций кнопок

Основано на [project.md](./project.md).

## Цель

По URL франкоязычной статьи приложение должно:
1. распарсить страницу;
2. по нажатию нужной кнопки вызвать AI;
3. показать **готовый текст** в блоке «Результат».

Кнопки из project.md:
- **«О чем статья?»** — краткое содержание на русском;
- **«Пост для Дзен»** — текст для публикации в Дзен;
- **«Пост для Telegram»** — текст для публикации в Telegram.

## Общий порядок работ

### Этап 1. Общая инфраструктура AI

1. **Вынести общий вызов OpenRouter** из `translateArticle` в функцию `callOpenRouter(prompt, options?)` в `lib/openrouter.ts` — один HTTP-клиент, одна модель DeepSeek, обработка ошибок.
2. **Описать промпты** для каждого действия в `lib/prompts.ts` (или рядом с `openrouter.ts`):
   - `buildSummaryPrompt(article)`
   - `buildDzenPrompt(article)`
   - `buildTelegramPrompt(article)`
3. **Добавить функции генерации** в `lib/openrouter.ts`:
   - `summarizeArticle(article)`
   - `generateDzenPost(article)`
   - `generateTelegramPost(article)`
4. **Обновить `lib/actions.ts`** — тексты загрузки для каждой кнопки (например: «Готовлю краткое содержание...»).

### Этап 2. Подключение в API

5. **Расширить `app/api/generate/route.ts`** — после парсинга вызывать нужную функцию по `action`:
   - `summary` → `summarizeArticle(parsed)`
   - `dzen` → `generateDzenPost(parsed)`
   - `telegram` → `generateTelegramPost(parsed)`
6. **Единый формат ответа** — `{ result: string, action, parsed? }`; в UI всегда выводить `result` как текст, не JSON.

### Этап 3. Промпты и требования к результату

7. **«О чем статья?»** (`summary`):
   - вход: заголовок, дата, основной текст (французский);
   - задача AI: 3–5 абзацев на русском — суть, ключевые факты, вывод;
   - без полного перевода, без markdown-служебных блоков.

8. **«Пост для Дзен»** (`dzen`):
   - вход: распарсенная статья;
   - задача AI: связный пост на русском — цепляющий заголовок, лид, основная часть, мягкий вывод;
   - стиль: информативный, для широкой аудитории; длина ~800–1500 слов (уточнить по тестам).

9. **«Пост для Telegram»** (`telegram`):
   - вход: распарсенная статья;
   - задача AI: короткий пост на русском — суть в 2–4 абзацах, можно эмодзи умеренно;
   - лимит ~1000–1500 символов, удобно читать с телефона.

### Этап 4. UI и UX

10. **Обновить `ArticleProcessor.tsx`** — при необходимости подписи кнопок и сообщения загрузки из `ACTION_LOADING_LABELS`.
11. **Проверить отображение** длинных текстов в блоке «Результат» (переносы, прокрутка).

### Этап 5. Конфигурация и деплой

12. **Переменные окружения** — `OPENROUTER_API_KEY` локально (`.env.local`) и на Vercel; при необходиости `OPENROUTER_BASE_URL`, модель в `.env.example`.
13. **Ограничение длины текста** — как у перевода: обрезка `content` до ~12 000 символов с пометкой в промпте, если статья очень длинная.

### Этап 6. Проверка

14. **Тесты вручную** на 2–3 реальных URL (новостной сайт, блог, длинная статья):
    - каждая кнопка возвращает осмысленный русский текст;
    - ошибки парсинга и API отображаются в UI;
    - время ожидания приемлемое (индикатор загрузки).
15. **Сверка с project.md** — три кнопки из ТЗ работают через AI; результат — готовый текст, не JSON.

## Рекомендуемая последовательность реализации

```
1. Рефакторинг openrouter (общий callOpenRouter)
2. Промпт + функция «О чем статья?»     → проверка end-to-end
3. Промпт + функция «Пост для Дзен»     → проверка end-to-end
4. Промпт + функция «Пост для Telegram» → проверка end-to-end
5. API route: все action → AI
6. UI: тексты загрузки, финальная проверка
7. Деплой на Vercel с OPENROUTER_API_KEY
```
